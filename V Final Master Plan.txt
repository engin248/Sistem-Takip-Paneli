Z3 TIMEOUT DAVRANIŞI:

Z3 solve timeout = 10 saniye
T+0s     Z3.solve() başlatıldı
T+10s    TIMEOUT → aşağıdaki sıra izlenir:
  Adım 1: Cache kontrol (proof cache'de benzer var mı?)
           → HIT → Z3 quick verify → kullan
           → MISS → Adım 2'ye geç
  Adım 2: Constraint sadeleştirme
           → Spec basitleştirilir → Z3 tekrar denenir (5s)
           → BAŞARILI → kullan
           → TIMEOUT → Adım 3'e geç
  Adım 3: DEGRADED MODE
           → İşlem "STRICT → SAFE" moduna düşürülür
           → Sadece kritik 32 kriter + cache verify
           → İşlem "degraded" olarak etiketlenir
           → İnsan bilgilendirilir (alert seviye 3)
  Z3 tamamen çöktüyse (process crash):
           → F4: ISOLATE
           → K5 modülü yeniden başlatılır (watchdog)
           → Tüm bekleyen işlemler kuyruğa alınır
           → 3 deneme → başarısız → F5: SHUTDOWN + insan escalation
⚠️ Z3 ASLA bypass edilmez.
   Timeout = proof yok → işlem bekler veya reddedilir.
   Hiçbir işlem "proof olmadan geçti" olarak işaretlenemez.
PROOF CACHE DÜZELTMESİ:

cache hit
  → hash kontrolü (SHA-256)
  → input/context match (aynı girdi mi?)
  → Z3 quick verify (hâlâ geçerli mi?)
  → kullan
⚠️ Cache poisoning önlendi:
  Cache key = SHA-256(input + OPA_policy_version + Z3_version)
  Kural değişikliği → policy version değişir → cache miss (SALT)
  Aksiyom değişikliği → TÜM cache temizle
  Z3 güncelleme → Z3_version değişir → cache miss (SALT)
